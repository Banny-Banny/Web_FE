사전분석단계)
1. package.json 확인하여 사용 가능한 라이브러리 분석
2. 피그마 구조 전체 분석 (레이아웃 → 섹션 → 컴포넌트 순)
3. globals.css 또는 Tailwind 설정에서 CSS 변수 색상 토큰 목록 확인
4. globals.css 또는 Tailwind 설정에서 CSS 변수 타이포그래피 토큰 목록 확인
5. 공통 컴포넌트 목록 확인 및 재사용 가능 여부 판단
   - src/commons/components/button: 버튼 컴포넌트 사용 가능 (필요시)
   - 현재는 독립 구현, 향후 공통 컴포넌트 추출 고려

==============================================

구현단계)
1. index.tsx: JSX 구조만 작성 (div, span 등 기본 HTML 요소 사용)
2. styles.module.css: CSS Modules로 모든 스타일 선언
3. 피그마 디자인과 1:1 대응되도록 레이아웃 구현
4. 토큰 기반 색상/타이포그래피만 사용 (CSS 변수 또는 Tailwind 토큰)
5. @remixicon/react의 아이콘 컴포넌트 사용 (필요시)
6. **소수점 값 반올림**: 모든 스타일 속성의 소수점 값은 반올림하여 정수로 변환
7. 버튼이 필요한 경우 Button 공통 컴포넌트 사용 (직접 구현 금지)
8. 색상/타이포그래피는 CSS 변수(var(--*)) 또는 Tailwind 토큰 사용
9. Button은 @/commons/components/button에서 import하여 사용
10. Props로 onSelectKakao, onSelectEmail 핸들러를 받아서 각 버튼 클릭 시 호출
