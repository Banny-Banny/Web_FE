==============================================

조건-API) 읽지 않은 알림 개수
  - 엔드포인트: GET /api/me/notifications/unread-count
  - 설명: 로그인한 사용자의 읽지 않은 알림 개수 조회
  - 응답: { count: number } 또는 { data: { count: number } }
  - 에러: 401 인증 필요

조건-API) 알림 목록 조회
  - 엔드포인트: GET /api/me/notifications
  - 설명: 사용자 알림 목록 조회 (unread/read 구분 또는 items+isRead)
  - 응답: GetNotificationsResponseData (unread[], read[] 또는 items[])
  - 에러: 401

조건-API) 알림 읽음 처리
  - 엔드포인트: POST /api/me/notifications/{notificationId}/read
  - 설명: 해당 알림을 읽음으로 처리
  - 에러: 401, 404 등

조건-API) 알림 삭제
  - 엔드포인트: POST /api/me/notifications/{notificationId}/delete
  - 설명: 해당 알림 삭제
  - 에러: 401, 404 등

조건-파일경로) CSS: src/components/Mypage/components/notification/styles.module.css

조건-API구현) endpoints.ts에 ME_NOTIFICATIONS_UNREAD_COUNT, ME_NOTIFICATIONS, ME_NOTIFICATION_READ(id), ME_NOTIFICATION_DELETE(id) 반영
조건-API구현) apiClient로 GET/POST 요청
조건-API구현) Notification, GetNotificationsResponseData 타입 사용. 응답이 unread/read 또는 items 형태 모두 처리

조건-ReactQuery) useNotifications: queryKey ['notifications'], unread/read 분리 반환, staleTime/gcTime 적절히 설정
조건-ReactQuery) useMarkNotificationRead, useDeleteNotification: mutation 성공 시 notifications 쿼리 무효화
조건-ReactQuery) useUnreadNotificationCount: queryKey ['notifications', 'unread-count'], 마이페이지 헤더 배지용

조건-컴포넌트구현) Notification: useNotifications, useMarkNotificationRead, useDeleteNotification 사용. 새 알림 클릭 시 getNotificationRoute(type, targetId, id)로 path 얻어 router.push(path), onClose(), markRead.mutate(id). 이전 알림 삭제 시 deleteNotification.mutate(id), useToast로 성공/실패
조건-컴포넌트구현) NotificationListContent: unread, read, isLoading, isError, error, refetch, onNewNotificationClick, onDeleteClick, deletingId, onMarkAllRead Props. 로딩·에러·빈 목록·새 알림/이전 알림 섹션·모두읽음·날짜 포맷(formatNotificationDate) 처리. 알림 타입(item.type)별 아이콘 매핑(SYSTEM→RiInformationLine, TIME_CAPSULE_INVITE→RiTimeLine, WAITING_ROOM_INVITE→RiMailSendLine, FRIEND_REQUEST→RiUserAddLine, NOTICE→RiMegaphoneLine, MY_EGGS→RiGiftLine, PROFILE→RiUserLine, 그 외→RiNotificationLine)
조건-공통컴포넌트) src/commons/components/ 활용 (Button, Spinner 등)
조건-색상토큰) CSS 변수 또는 Tailwind 토큰만 사용 (하드코딩 금지)
조건-아이콘/이미지) @remixicon/react 또는 src/assets/ 사용, 외부 아이콘 패키지 금지

핵심요구사항) GET /api/me/notifications/unread-count로 읽지 않은 알림 개수 조회 (useUnreadNotificationCount, 마이페이지 헤더 배지용)
핵심요구사항) GET /api/me/notifications로 알림 목록 조회, useNotifications로 unread/read 분리 표시

핵심요구사항) POST /api/me/notifications/{id}/read 읽음 처리, POST /api/me/notifications/{id}/delete 삭제 처리. 삭제 시 useToast로 결과 메시지
핵심요구사항) 로딩·에러·빈 목록·다시 시도·모두읽음·이전 알림 삭제 버튼 처리
