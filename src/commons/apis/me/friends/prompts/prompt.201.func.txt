==============================================

조건-API) 카카오 친구 목록 연동 (동기화)
  - 엔드포인트: POST /api/auth/kakao/friends-sync
  - 설명: 카카오에서 친구 목록을 가져와 서버 DB에 반영. 친구 목록 조회 전 호출 시 최신 카카오 친구가 반영된 목록을 받을 수 있음
  - 요청: 본문 없음 또는 빈 객체 (인증 토큰으로 사용자 식별)
  - 응답: 성공 시 200 등 (스펙에 따름)
  - 에러: 401 인증 필요, 카카오 미연동 사용자 등 실패 시에도 목록 조회는 정상 진행할 것

조건-API) 친구 목록 조회
  - 엔드포인트: GET /api/me/friends
  - 설명: 로그인한 사용자의 친구 목록을 페이지네이션으로 조회 (동기화 반영 후 조회)
  - 파라미터: limit (선택, 기본 20), offset (선택, 기본 0)
  - 응답: GetFriendsResponse (items, total, limit, offset, hasNext 등)
  - 에러: 401 인증 필요

조건-파일경로) 엔드포인트: src/commons/apis/endpoints.ts (AUTH_ENDPOINTS.KAKAO_FRIENDS_SYNC, ME_FRIENDS)
조건-파일경로) API 함수: src/commons/apis/me/friends/index.ts (syncKakaoFriends, getFriends, addFriend, deleteFriend)
조건-파일경로) API 타입: src/commons/apis/me/friends/types.ts
조건-파일경로) React Query 훅: src/commons/apis/me/friends/hooks/useFriends.ts

==============================================

조건-API구현) endpoints.ts에 KAKAO_FRIENDS_SYNC 추가 (값: /api/auth/kakao/friends-sync)
조건-API구현) syncKakaoFriends: apiClient.post(AUTH_ENDPOINTS.KAKAO_FRIENDS_SYNC) 호출
조건-API구현) syncKakaoFriends 실패 시 예외를 던지지 않고 무시 (try/catch에서 catch 시 아무 처리 없이 종료). 카카오 미연동·네트워크 오류 등으로 실패해도 친구 목록 조회는 진행
조건-API구현) getFriends는 기존대로 GET /api/me/friends 호출, GetFriendsParams/GetFriendsResponse 타입 사용

조건-ReactQuery) useFriends의 queryFn: 먼저 await syncKakaoFriends() 호출 후 getFriends(params) 반환
조건-ReactQuery) queryKey, enabled, staleTime, gcTime, retry 정책은 기존 useFriends 유지
조건-ReactQuery) 친구 목록을 불러오는 모든 경로에서 위 useFriends 사용 시 자동으로 연동 후 조회되도록 함

==============================================

기존히스토리-참고만) 온보딩 완료 시 friend_consent로 친구 연동 동의를 서버에 전송 (POST /api/onboarding/complete)
기존히스토리-참고만) 친구 목록 조회는 GET /api/me/friends로만 하면 DB에 저장된 목록만 반환. 카카오 친구를 반영하려면 friends-sync 호출 후 조회 필요
기존히스토리-참고만) useFriends는 enabled 옵션으로 친구 연동 미동의 시 호출 생략 가능 (예: profile?.friendConsent === true)

==============================================

핵심요구사항) 친구 목록을 불러올 때 실제 카카오 친구 연동이 이루어지도록 연결
핵심요구사항) useFriends 호출 시 queryFn 내부에서 1) syncKakaoFriends() 2) getFriends(params) 순서로 실행
핵심요구사항) syncKakaoFriends 실패 시에도 getFriends는 정상 수행 (기존 DB 친구 목록이라도 표시)
핵심요구사항) 엔드포인트·API 함수·훅이 기존 코드 스타일 및 에러 처리 규칙과 일치
