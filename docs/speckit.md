# 🛠 Spec Kit 기반 AI 협업 가이드

## 📋 개요

**이 프로젝트는 GitHub Spec-Kit 워크플로우를 사용합니다.**

우리 팀은 GitHub Spec-Kit 워크플로우를 사용하여 AI와 함께 체계적으로 기능을 개발합니다. 모든 기능 구현은 아래 4단계를 엄격히 따릅니다.

---

## 🚀 개발 프로세스

### 1️⃣ 사양 정의 (Specification)
- **명령어**: `/speckit.specify`
- **목적**: 
  - **'무엇'을 만들 것인지** 명확히 정의합니다
  - **'왜' 만들어야 하는지** 비즈니스 목적과 사용자 가치를 문서화합니다
  - 기능의 범위와 제약사항을 명확히 합니다
- **활동**: 
  - AI가 생성한 `spec.md`를 검토하며 사용자 요구사항이 충족되었는지 확인합니다
  - 팀원들과 사양을 공유하고 피드백을 반영합니다

### 2️⃣ 기술 계획 (Technical Plan)
- **명령어**: `/speckit.plan`
- **목적**: 
  - **기술 스택**을 선택하고 근거를 명시합니다
  - **아키텍처 패턴**과 폴더 구조를 설계합니다
  - 프로젝트 헌법에 정의된 규칙이 반영되었는지 검증합니다
- **활동**: 
  - 프로젝트 규칙(Next.js, Pretendard 폰트, 블랙&화이트 미니멀 디자인 등)이 반영된 `plan.md`를 확정합니다
  - 기술적 의사결정의 근거를 문서화합니다

### 3️⃣ 작업 분해 (Task Breakdown)
- **명령어**: `/speckit.tasks`
- **목적**: 
  - 설계를 **구현 가능한 작은 단위(Task)**로 쪼갭니다
  - 각 Task가 **독립적으로 테스트 가능한 수준**으로 분해되었는지 확인합니다
  - 작업 간 의존성을 명확히 합니다
- **활동**: 
  - Task 목록을 검토하고 우선순위를 정합니다
  - 각 Task의 완료 기준을 명확히 합니다

### 4️⃣ 실제 구현 (Implementation)
- **명령어**: `/speckit.implement` (또는 에이전트에게 직접 지시)
- **목적**: 
  - 확정된 Task를 바탕으로 **실제 코드를 작성**합니다
  - 프로젝트 헌법의 규칙을 준수하며 구현합니다
  - 코드 리뷰와 테스트를 통해 품질을 보장합니다
- **활동**: 
  - AI가 작성한 코드를 리뷰하고 빌드 오류를 함께 수정합니다
  - 프로젝트 규칙 준수 여부를 확인합니다

## 📜 프로젝트 헌법 (Core Rules)

프로젝트 헌법(`/constitution`)에 정의된 우리의 핵심 규칙을 반드시 준수해야 합니다:

### ✅ 필수 기술 스택
- **Next.js**: 모든 페이지와 라우팅은 Next.js 프레임워크를 사용합니다
- **Pretendard 폰트**: 일관된 타이포그래피를 위해 Pretendard 폰트를 적용합니다

### 🎨 디자인 원칙
- **블랙 & 화이트 미니멀 디자인**: 
  - 색상은 블랙과 화이트를 기본으로 사용합니다
  - 불필요한 장식 요소를 제거하고 미니멀한 디자인을 유지합니다
  - 디자인 토큰은 `commons/constants/`에서 관리하며 하드코딩을 금지합니다

> ⚠️ **중요**: 모든 개발 단계에서 위 규칙들이 반영되었는지 확인하세요. `/speckit.plan` 단계에서 특히 주의 깊게 검토해야 합니다.

## 🎯 적용 예시: Easter Egg 프로젝트

**팀 프로젝트 'Easter Egg'의 관리자 페이지(admin) 개발에 이 가이드를 적극 활용하세요.**

관리자 페이지 개발 시:
1. `/speckit.specify`로 관리자 페이지의 기능과 요구사항을 정의합니다
2. `/speckit.plan`으로 Next.js 기반의 관리자 페이지 아키텍처를 설계합니다
3. `/speckit.tasks`로 작업을 세분화합니다 (예: 사용자 관리, 콘텐츠 관리, 통계 대시보드 등)
4. `/speckit.implement`로 실제 구현을 진행하며 Pretendard 폰트와 블랙&화이트 미니멀 디자인을 적용합니다

각 단계에서 생성된 문서(`spec.md`, `plan.md`, `tasks.md`)는 팀원들과 공유하여 일관된 개발 프로세스를 유지하세요.
