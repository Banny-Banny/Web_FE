# TimeEgg 웹 레이아웃 및 GNB(Global Navigation Bar) 명세서

## 📋 개요

### 프로젝트 목표
TimeEgg 웹 애플리케이션의 기본 레이아웃 구조를 구축하고, 사용자가 주요 기능에 쉽게 접근할 수 있는 전역 네비게이션 바를 제공합니다.

### 비즈니스 가치
- **사용자 경험 향상**: 주요 기능(소식, 홈, 마이페이지)에 대한 직관적이고 빠른 접근 제공
- **일관성**: 모든 페이지에서 동일한 네비게이션 경험으로 사용자 혼란 최소화
- **모바일 최적화**: 모바일 환경을 기준으로 한 고정 레이아웃으로 다양한 기기에서 일관된 경험 제공
- **컨텍스트 인식**: 인증 관련 페이지에서는 네비게이션을 숨겨 사용자 집중도 향상

### 범위
- **포함**: 기본 레이아웃 구조, GNB 컴포넌트, 페이지별 GNB 표시/숨김 로직, 모바일 기준 고정 레이아웃
- **제외**: GNB 내 각 메뉴의 상세 페이지 구현, 알림 기능 구현, 사용자 프로필 기능 구현

## 🎯 사용자 시나리오

### US1: 인증되지 않은 사용자의 레이아웃 경험

**시나리오**: 사용자가 로그인, 회원가입, 온보딩 페이지에 접근할 때

**사용자 행동**:
1. 사용자가 로그인 페이지에 접근
2. 사용자가 회원가입 페이지에 접근
3. 사용자가 온보딩 페이지에 접근

**기대 결과**:
- 모든 인증 관련 페이지에서 GNB가 표시되지 않음
- 페이지 콘텐츠에만 집중할 수 있는 깔끔한 화면 제공
- 레이아웃은 모바일 기준 고정값으로 일관되게 유지됨

**성공 기준**:
- 로그인, 회원가입, 온보딩 페이지에서 GNB 표시 여부가 0% (완전히 숨김)
- 레이아웃 너비가 모바일 기준 고정값으로 일관되게 유지됨

### US2: 인증된 사용자의 주요 기능 접근

**시나리오**: 사용자가 로그인 후 메인 기능 페이지에 접근할 때

**사용자 행동**:
1. 사용자가 홈 페이지에 접근
2. 사용자가 소식 페이지에 접근하려고 GNB의 소식 아이콘을 클릭
3. 사용자가 마이페이지에 접근하려고 GNB의 마이 아이콘을 클릭
4. 사용자가 다른 페이지에서 홈으로 돌아가려고 GNB의 홈 아이콘을 클릭

**기대 결과**:
- 화면 하단에 GNB가 항상 표시됨
- 소식, 홈, 마이 아이콘이 왼쪽에서 오른쪽 순서로 배치됨
- 각 아이콘을 클릭하면 해당 페이지로 이동함
- 현재 페이지에 해당하는 아이콘이 시각적으로 구분됨 (선택적)

**성공 기준**:
- 인증된 사용자가 접근 가능한 모든 페이지에서 GNB 표시율 100%
- GNB 아이콘 클릭 시 해당 페이지로 이동 성공률 100%
- 아이콘 배치 순서가 왼쪽에서 오른쪽으로 (소식 → 홈 → 마이) 100% 일치

### US3: 모바일 환경에서의 레이아웃 경험

**시나리오**: 사용자가 모바일 기기에서 웹 애플리케이션을 사용할 때

**사용자 행동**:
1. 사용자가 모바일 브라우저에서 애플리케이션에 접근
2. 사용자가 다양한 모바일 기기(다양한 화면 크기)에서 접근

**기대 결과**:
- 레이아웃이 모바일 기준 고정값으로 일관되게 표시됨
- GNB가 화면 하단에 고정되어 항상 접근 가능함
- 콘텐츠 영역이 GNB와 겹치지 않도록 적절한 여백 유지됨

**성공 기준**:
- 모든 모바일 기기에서 레이아웃 너비가 고정값으로 일관되게 표시됨
- GNB가 화면 하단에 고정되어 스크롤과 무관하게 항상 표시됨
- 콘텐츠 영역과 GNB 간 겹침 발생률 0%

## 📋 기능 요구사항

### FR-001: 기본 레이아웃 구조
**설명**: 웹 애플리케이션의 기본 레이아웃 컨테이너를 모바일 기준 고정값으로 구성합니다.

**요구사항**:
- 레이아웃 너비는 모바일 기준 고정값으로 설정됨
- 콘텐츠 영역이 중앙에 배치됨
- 좌우 여백이 일관되게 유지됨

**검증 기준**:
- 레이아웃 너비가 모바일 기준 고정값으로 설정되어 있음
- 모든 페이지에서 레이아웃 너비가 일관되게 유지됨
- 콘텐츠가 중앙 정렬되어 표시됨

### FR-002: GNB 컴포넌트 구성
**설명**: 화면 하단에 고정되는 전역 네비게이션 바를 구성합니다.

**요구사항**:
- GNB는 화면 하단에 고정되어 표시됨
- GNB 내부에 3개의 메뉴 아이콘이 왼쪽에서 오른쪽 순서로 배치됨
  - 소식 (알림 아이콘)
  - 홈 (지도 아이콘)
  - 마이 (사람 아이콘)
- 각 아이콘은 클릭 가능하며 해당 페이지로 이동함
- 아이콘은 lucide-react 라이브러리를 사용하여 표시됨

**검증 기준**:
- GNB가 화면 하단에 고정되어 스크롤과 무관하게 표시됨
- 아이콘 배치 순서가 왼쪽에서 오른쪽으로 (소식 → 홈 → 마이) 정확히 일치함
- 각 아이콘 클릭 시 해당 페이지로 이동함
- 아이콘이 lucide-react 라이브러리로 렌더링됨

### FR-003: 페이지별 GNB 표시 제어
**설명**: 특정 페이지(로그인, 회원가입, 온보딩)에서는 GNB를 숨깁니다.

**요구사항**:
- 로그인 페이지에서 GNB가 표시되지 않음
- 회원가입 페이지에서 GNB가 표시되지 않음
- 온보딩 페이지에서 GNB가 표시되지 않음
- 그 외 모든 페이지에서 GNB가 표시됨

**검증 기준**:
- 로그인 페이지에서 GNB 표시 여부가 0% (완전히 숨김)
- 회원가입 페이지에서 GNB 표시 여부가 0% (완전히 숨김)
- 온보딩 페이지에서 GNB 표시 여부가 0% (완전히 숨김)
- 그 외 페이지에서 GNB 표시 여부가 100%

### FR-004: 모바일 기준 고정 레이아웃
**설명**: 레이아웃이 모바일 기준 고정값으로 설정되어 다양한 기기에서 일관된 경험을 제공합니다.

**요구사항**:
- 레이아웃 너비가 모바일 기준 고정값으로 설정됨
- 데스크톱 환경에서도 모바일 기준 고정값이 유지됨
- 좌우 여백이 일관되게 유지됨

**검증 기준**:
- 모든 기기에서 레이아웃 너비가 모바일 기준 고정값으로 일관되게 표시됨
- 데스크톱 환경에서도 모바일 기준 고정값이 유지됨
- 좌우 여백이 모든 페이지에서 일관되게 유지됨

## 🚀 비기능 요구사항 (NFR)

### 성능
- **NFR-001**: GNB 렌더링 시간이 페이지 로드 시간에 미치는 영향이 5% 이하

- **NFR-004**: 키보드 네비게이션으로 GNB 아이콘에 접근 가능함
- **NFR-005**: 아이콘의 최소 터치 타겟 크기가 44px × 44px 이상

### 사용자 경험
- **NFR-006**: GNB가 화면 하단에 고정되어 항상 접근 가능함
- **NFR-007**: 콘텐츠 영역과 GNB 간 겹침이 발생하지 않음
- **NFR-008**: 현재 페이지에 해당하는 아이콘이 시각적으로 구분됨 (선택적, 향후 구현)

### 호환성
- **NFR-009**: 주요 모바일 브라우저(Chrome, Safari, Firefox)에서 정상 동작
- **NFR-010**: 다양한 모바일 기기 화면 크기에서 레이아웃이 일관되게 표시됨

## 🔍 엣지 케이스 및 예외 상황

### EC-001: 페이지 전환 중 GNB 상태
**상황**: 사용자가 한 페이지에서 다른 페이지로 이동하는 중

**처리 방법**: 
- 페이지 전환 중에도 GNB가 계속 표시되어야 함 (로딩 상태와 무관)
- 페이지 전환 완료 후에도 GNB 상태가 유지됨

### EC-002: 매우 작은 화면 크기
**상황**: 화면 크기가 매우 작은 모바일 기기에서 접근

**처리 방법**:
- 레이아웃 너비가 모바일 기준 고정값으로 유지됨
- GNB 아이콘 크기가 최소 터치 타겟 크기(44px × 44px)를 유지함
- 콘텐츠 영역이 GNB와 겹치지 않도록 적절한 여백 유지됨

### EC-003: 인증 상태 불명확
**상황**: 사용자의 인증 상태를 확인할 수 없는 경우

**처리 방법**:
- 기본적으로 GNB를 표시함
- 인증이 필요한 페이지에서는 인증 상태 확인 후 GNB 표시 여부 결정

### EC-004: 아이콘 라이브러리 로딩 실패
**상황**: lucide-react 라이브러리 로딩에 실패한 경우

**처리 방법**:
- 대체 아이콘 또는 텍스트 라벨을 표시함
- 사용자가 기능을 계속 사용할 수 있도록 함

## 🎨 디자인 제약사항

### 레이아웃
- **모바일 기준 고정값**: 레이아웃 너비는 모바일 기준 고정값으로 설정됨
- **중앙 정렬**: 콘텐츠가 화면 중앙에 배치됨
- **일관성**: 모든 페이지에서 레이아웃 구조가 일관되게 유지됨

### GNB 디자인
- **하단 고정**: GNB는 화면 하단에 고정되어 표시됨
- **아이콘 배치**: 왼쪽에서 오른쪽 순서로 (소식 → 홈 → 마이) 배치됨
- **아이콘 라이브러리**: lucide-react를 사용하여 아이콘 표시
- **터치 타겟**: 각 아이콘의 최소 터치 타겟 크기는 44px × 44px 이상

### 색상 및 스타일
- **디자인 토큰 준수**: 프로젝트의 디자인 토큰 시스템을 준수함
- **일관성**: 기존 디자인 시스템과 일관된 스타일 적용
