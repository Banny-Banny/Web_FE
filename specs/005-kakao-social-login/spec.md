# TimeEgg 카카오 소셜 로그인 기능 명세서

## 📋 개요

### 프로젝트 목표
TimeEgg 웹 애플리케이션에서 사용자가 카카오 계정을 사용하여 간편하게 로그인할 수 있는 소셜 로그인 기능을 제공합니다.

### 비즈니스 가치
- **사용자 편의성**: 별도의 계정 생성 없이 카카오 계정으로 즉시 서비스 이용 가능
- **로그인 장벽 완화**: 비밀번호 기억 부담 없이 빠르고 간편한 로그인 경험 제공
- **사용자 유입 증가**: 소셜 로그인을 통한 회원가입 전환율 향상 기대
- **사용자 경험**: 로그인 후 자동으로 인증된 상태로 애플리케이션 주요 기능에 접근 가능

### 범위
- **포함**: 카카오 로그인 버튼 UI, 카카오 인증 플로우, 인증 토큰 저장, 로그인 성공/실패 처리, 기존 자체 로그인과의 통합
- **제외**: 다른 소셜 로그인 제공자(구글, 네이버 등), 카카오 계정 연동 해제 기능, 카카오 프로필 정보 수정 기능

## 🎯 사용자 시나리오

### US1: 카카오 계정으로 로그인 성공 (신규 사용자)

**시나리오**: 카카오 계정을 보유한 신규 사용자가 카카오 소셜 로그인을 통해 처음 서비스에 가입하고 로그인할 때

**사용자 행동**:
1. 사용자가 로그인 페이지에 접근
2. 사용자가 "카카오로 시작하기" 버튼을 클릭
3. 사용자가 카카오 로그인 팝업에서 카카오 계정 정보를 입력하고 로그인
4. 사용자가 카카오 서비스 이용 동의 화면에서 필요한 권한에 동의

**기대 결과**:
- 카카오 인증이 완료됨
- 서버에서 사용자 계정이 자동으로 생성되거나 기존 계정과 연동됨
- 인증 토큰이 발급되어 안전하게 저장됨
- 사용자가 애플리케이션의 주요 기능 페이지로 자동 이동됨
- 사용자가 인증된 상태로 애플리케이션을 사용할 수 있음

**성공 기준**:
- 카카오 로그인 성공률이 95% 이상 (유효한 카카오 계정 제공 시)
- 카카오 인증부터 주요 기능 페이지 이동까지의 시간이 5초 이하
- 신규 사용자 계정 자동 생성 성공률 100%
- 인증 토큰이 안전하게 저장되어 이후 요청에서 사용 가능

## 📋 기능 요구사항

### FR-001: 카카오 로그인 버튼 제공
**설명**: 사용자가 카카오 소셜 로그인을 시작할 수 있는 버튼을 제공합니다.

**요구사항**:
- `LoginMethodSelector` 컴포넌트(`src/components/Login/LoginMethodSelector/index.tsx`)에 "카카오로 시작하기" 버튼이 표시됨
- 버튼이 `onSelectKakao` prop을 통해 `LoginContainer`의 `handleSelectKakao` 함수와 연결됨
- 버튼 디자인이 제공된 Figma 디자인과 일치함
- 버튼이 명확하게 식별 가능하고 클릭하기 쉬운 크기로 표시됨
- 버튼 클릭 시 `src/components/Login/index.tsx`의 `handleSelectKakao` 함수가 실행되어 카카오 로그인 플로우가 시작됨

**검증 기준**:
- 카카오 로그인 버튼이 로그인 방법 선택 화면에 표시됨
- 버튼이 올바른 핸들러 함수와 연결됨
- 버튼 디자인이 Figma 디자인과 시각적으로 일치함
- 버튼 클릭 시 카카오 로그인 플로우가 시작됨

### FR-002: 카카오 인증 플로우 실행
**설명**: 사용자가 카카오 계정으로 인증할 수 있는 플로우를 제공합니다.

**요구사항**:
- 카카오 로그인 버튼 클릭 시 `src/components/Login/hooks/useKakaoLogin.ts`의 `loginWithKakao()`가 실행됨
- 웹은 **앱과 동일한 OAuth 리다이렉트 방식**을 사용함 (카카오 JavaScript SDK 미사용)
- `window.location.href`로 백엔드 OAuth 시작 엔드포인트로 이동함: `GET /api/auth/kakao?redirect_uri=<frontend_callback_url>`
- 카카오 인증 완료 후, 백엔드가 프론트 콜백으로 리다이렉트함: `/auth/callback?token=...`

**검증 기준**:
- 카카오 로그인 버튼 클릭 시 백엔드 OAuth 시작 엔드포인트로 리다이렉트됨
- 인증 완료 후 `/auth/callback?token=...`로 진입하여 토큰 처리 로직이 실행됨

### FR-003: 서버 인증 및 토큰 발급
**설명**: 카카오 인증 정보를 서버로 전송하여 애플리케이션 인증 토큰을 받습니다.

**요구사항**:
- 프론트는 백엔드 OAuth 흐름을 시작하기 위해 `GET /api/auth/kakao?redirect_uri=...`로 이동함
- 백엔드가 카카오 인증을 처리한 뒤, 프론트 콜백(`/auth/callback`)로 `token`을 전달하여 리다이렉트함
- 콜백 페이지(`src/app/(auth)/auth/callback/page.tsx`)에서 `token`을 `saveTokens()`로 저장함
- 저장된 토큰으로 `AUTH_ENDPOINTS.VERIFY`를 호출(`verifyAuth()`)하여 사용자 정보를 갱신함

**검증 기준**:
- `/auth/callback?token=...` 진입 시 토큰이 저장됨
- 토큰 검증 API 호출로 사용자 정보가 캐시에 저장됨

### FR-004: 로그인 성공 처리
**설명**: 카카오 로그인에 성공하면 사용자를 적절한 페이지로 이동시킵니다.

**요구사항**:
- `src/components/Login/hooks/useKakaoLoginMutation.ts`의 `useKakaoLoginMutation` 훅을 사용하여 로그인 처리
- `useMutation` 훅을 사용하여 비동기 로그인 요청 처리
- 카카오 로그인 성공 시 `onSuccess` 콜백에서 인증 토큰 저장 및 사용자 정보 업데이트
- `queryClient.setQueryData(['auth', 'user'], data.user)`를 통해 사용자 정보를 React Query 캐시에 저장
- 온보딩 완료 여부를 `queryClient.getQueryData(['onboarding', 'status'])`로 확인
- 신규 사용자 또는 온보딩 미완료 사용자의 경우 `/onboarding` 페이지로 리다이렉트
- 기존 사용자 또는 온보딩 완료 사용자의 경우 `/` (홈) 페이지로 리다이렉트
- 사용자가 인증된 상태로 애플리케이션을 사용할 수 있음

**검증 기준**:
- 카카오 로그인 성공 시 인증 토큰이 저장됨
- 사용자 정보가 React Query 캐시에 저장됨
- 온보딩 상태에 따라 적절한 페이지로 자동 이동됨
- 인증된 상태가 애플리케이션 전역에서 유지됨

### FR-005: 로그인 실패 처리
**설명**: 카카오 로그인에 실패하면 사용자에게 적절한 오류 메시지를 표시합니다.

**요구사항**:
- `useKakaoLoginMutation` 훅의 `onError` 콜백에서 에러 처리
- 카카오 인증 실패 시 사용자에게 명확한 오류 메시지 표시
- 서버 연동 실패 시 `apiClient.post()`에서 발생한 Axios 에러를 `ApiError` 형식으로 변환하여 처리
- 네트워크 오류 시 사용자에게 명확한 오류 메시지 표시
- 에러 처리 시 자체 로그인(`src/commons/apis/auth/login.ts`)과 동일한 패턴 사용
- 오류 메시지가 사용자가 이해하기 쉬운 언어로 표시됨
- 오류 발생 후에도 사용자가 다시 시도하거나 다른 로그인 방법을 선택할 수 있음

**검증 기준**:
- 로그인 실패 시 사용자에게 오류 메시지 표시
- 에러가 적절한 형식으로 변환되어 처리됨
- 오류 메시지가 상황에 맞게 적절히 표시됨
- 오류 발생 후에도 로그인 페이지가 정상적으로 동작함

### FR-006: 로그인 취소 처리
**설명**: 사용자가 카카오 로그인을 취소하면 조용히 처리합니다.

**요구사항**:
- OAuth 리다이렉트 플로우 특성상, 사용자가 카카오 인증 화면에서 취소하면 백엔드가 에러 상태로 콜백 리다이렉트할 수 있음
- 이 경우 프론트는 콜백에서 토큰 부재(또는 실패 응답)를 감지하여 일반적인 인증 실패로 처리하되, 사용자 경험 정책에 따라 조정 가능

**검증 기준**:
- (백엔드 동작 정의에 따라) 취소 시 로그인 페이지로 복귀하며, 과도한 경고/에러 노출이 없음

### FR-007: 기존 자체 로그인과의 통합
**설명**: 카카오 소셜 로그인과 기존 자체 로그인이 동일한 인증 시스템을 사용합니다.

**요구사항**:
- 카카오 로그인과 자체 로그인(`src/commons/apis/auth/login.ts`)이 동일한 인증 토큰 형식을 사용함
- 카카오 로그인과 자체 로그인이 동일한 `src/commons/utils/auth.ts`의 토큰 저장 함수를 사용함
- 카카오 로그인과 자체 로그인이 동일한 React Query 캐시 구조를 사용함
- `useKakaoLoginMutation` 훅이 `useLoginMutation` (`src/components/Login/hooks/useLoginMutation.ts`)과 동일한 패턴을 따름
- 카카오 로그인과 자체 로그인이 동일한 온보딩 확인 및 리다이렉트 로직을 사용함
- `LoginContainer` (`src/components/Login/index.tsx`)에서 `handleSelectKakao` 함수를 통해 카카오 로그인을 연결

**검증 기준**:

## 🔍 엣지 케이스 및 예외 상황

### EC-001: 카카오 로그인 중 페이지 이탈
**상황**: 사용자가 카카오 로그인 요청 중에 페이지를 이탈하는 경우

**처리 방법**:
- `useMutation`의 진행 중인 요청을 취소하거나 무시
- React Query의 `queryClient`에서 진행 중인 쿼리를 정리
- 사용자가 다시 로그인 페이지에 접근할 수 있음
- 사용자에게 오류 메시지가 표시되지 않음

### EC-002: 이미 인증된 사용자의 접근
**상황**: 이미 로그인한 사용자가 로그인 페이지에 접근하는 경우

**처리 방법**:
- `queryClient.getQueryData(['auth', 'user'])`로 인증 상태 확인
- 인증된 사용자의 경우 주요 기능 페이지로 자동 리다이렉트
- 또는 로그인 페이지에 접근할 수 있도록 허용 (선택적)

### EC-003: 토큰 저장 실패
**상황**: 카카오 로그인은 성공했지만 `saveTokens()` 함수 호출 시 토큰 저장에 실패한 경우

**처리 방법**:
- `useKakaoLoginMutation`의 `onSuccess`에서 토큰 저장 실패를 감지
- 사용자에게 오류 메시지 표시
- 사용자가 다시 로그인을 시도할 수 있도록 함

### EC-004: 서버 응답 지연
**상황**: 카카오 인증은 완료되었지만 서버 응답이 지연되는 경우

**처리 방법**:
- `apiClient.post()`에서 적절한 타임아웃 설정
- `useMutation`의 `onError`에서 타임아웃 에러를 감지
- 타임아웃 발생 시 사용자에게 명확한 안내 제공
- 사용자가 다시 시도할 수 있도록 함

### EC-005: 카카오 인증 팝업 차단
**상황**: 사용자의 브라우저에서 팝업이 차단된 경우

**처리 방법**:
- `Kakao.Auth.login()` 호출 시 팝업이 차단되면 에러를 감지
- 사용자에게 팝업 차단 해제 안내 제공
- 또는 카카오 SDK의 리다이렉트 방식으로 대체 고려

### EC-006: 카카오 계정 연동 오류
**상황**: 카카오 인증은 성공했지만 서버에서 계정 연동에 실패한 경우

**처리 방법**:
- `apiClient.post()`에서 받은 서버 응답의 에러를 `ApiError` 형식으로 변환
- `useKakaoLoginMutation`의 `onError`에서 에러를 처리
- 사용자에게 명확한 오류 메시지 표시
- 사용자가 다시 시도하거나 다른 로그인 방법을 선택할 수 있도록 함

### EC-007: 동일한 카카오 계정으로 다른 로그인 방법 사용
**상황**: 이미 카카오 계정으로 가입한 사용자가 동일한 이메일로 자체 로그인을 시도하는 경우

**처리 방법**:
- 서버에서 계정 상태를 확인하고 적절한 오류 메시지 표시
- 자체 로그인 API(`src/commons/apis/auth/login.ts`)에서 에러 응답 반환
- 사용자에게 카카오 로그인을 사용하도록 안내
